Lab 01 - Networks Creation

Virtual Environment Configuration

You are required to create two networks:

  • a NAT or NAT-Network for connecting your router to the internet
  • a Host-only Network for networking all your other VM

You can use VirtualBox, VMWare, or any other virtualization software that supports the creation and management of multiple networks. I recommend VirtualBox and provide instructions for this software only. You are responsible for implementing equivalent configurations in whatever virtualization software you choose to use.

  1. Create a new Host-only network in VirtualBox, and make note of its address space.
    • Do NOT enable DHCP since we will be using static IP addresses.
      • We will be running a VM as a router that will provide DHCP service on this network
    • I chose 192.168.60.0/24 as my network address. For this example, in VirtualBox, I set my adapter address to 192.168.60.254 with mask 255.255.255.0. If you choose a different IPv4 address, make note of it, because you will be assigning static IP addresses within this address space for all your following labs.
  2. Create a NAT-Network. You can also use the default NAT; you will have more visibility and some control over the IP address being assigned to your router. This will be used as a WAN interface for your router.

Install The Internet Gateway

OPNsense VM Creation

  • Download from OPNsense Download Page, choose AMD64, DVD, and your desired mirror for your download.
  • You will be downloading a compressed bz2 file. Extract this file to get the .iso DVD image file that can be used with your VM.
  • Create a new VM:
    • Choose an appropriate name, for example: NETS1037_OPNsense
    • You can select the OPNsense ISO image you have downloaded
    • Set your OS to BSD, and OS distribution to FreeBSD and version to FreeBSD (64-bit).
    • The recommended minimum requirements according to the documentation are:
      • at least 2 CPU cores
      • minimum of 3072 MB of base memory for the installer, but 4096 MB is recommended.
        • If you are seeking to reduce the memory footprint, you can reduce the memory after you have installed and enabled the required features. As an example for simple routing 1024 MB may be enough.
      • 40 GB storage for being able to enable all standard features
  • Modify the VM settings to connect to networks:
    • Under the Network settings for your VM, attach the Adapter 1 to the NAT or the NAT Network.
    • Enable the Adapter 2, and attach to your Host-only Adapter.

OPNsense Installation

Once your VM is started, it will boot from the .iso image and run in live mode. Wait for the login prompt. At this point, you are given two choices: to log in as root or to log in as installer.

  • Use installer as username and opnsense as password to start the TUI (text user interface) installation wizard
  • You can use the defaults for most settings. Install the software in the virtual storage device, and verify that you are OK with wiping the drive.
  • Make sure to change your root password to a password you can remember
  • Complete the installation and halt the system.
  • Wait for the VM to shut down, so you can detach the optical media (the .iso file) from the VM

OPNsense Configuration

Basic IP configuration

  • After your VM is powered off, detach the installation media from the VM. Power on the VM and wait for the system to boot
  • Log in as root and in the menu select option 1) Assign interfaces to set your WAN and LAN interfaces
    • Skip the LAGGs and VLAN configurations
    • Assign your WAN interface to be the interface that is attached to NAT/NAT-network.
    • Notice: If you cannot identify which interface is which, you can look at the MAC address displayed in OPNsense with the Adapters in your VM network adapter configuration
    • Set the interface that is connected to the Host-only network as your LAN.
  • Configure IP and DHCP server for your LAN interface by choosing 2) Set interface IP address.
    • Set Configure IPv4 address LAN interface via DHCP to N.
    • Provide your new LAN address (I chose 192.168.60.1 for my setup), and the subnet bit count to 24.
    • Do NOT set a gateway for your LAN interface
    • Enable the DHCP server on the LAN.
      • Set the DHCP range depending on your setup. As an example, I set mine between 192.168.60.100 and 192.168.60.200.
    • Keep the GUI web interface settings as HTTPS, and follow the prompts to generate a certificate and create default GUI settings

Web GUI Setup Wizard and Other Configuration

If everything is configured correctly, you should be able to access the web user interface by typing the IP address of your router in your host browser. Use your root credentials to log in.

Usually, the setup wizard will start the first time you log in. If you need to run the wizard, navigate to System > Wizard using the menus on the left.

  • Set the hostname for the device to opnsense.
  • Set your domain to home.apra.
  • Set your timezone to America/Toronto.
  • Make sure that the DNS resolver is enabled
  • On the next page, keep the Network (WAN) settings with default values:
    • Type is set to DHCP, and all other fields, such as MTU and MAC (spoofed), are left empty.
    • Make sure the default security settings are enabled: Block RFC1918 Private Networks and Block bogon networks.
  • Your LAN settings should already be configured correctly.
  • The Deployment Type defaults should be kept with Automatic DHCP/DNS registration as enabled
  • You can keep your root password or change it if desired
  • Finish the wizard

Add a DNS override for your host computer (i.e. your laptop) and your OPNsense router:

  • Navigate to Services > Unbound DNS > Overrides
  • Under Hosts use the + button to add two overrides for your router and your host laptop:
    • Set the hostname (for example, opnsense for your router and laptop for your host laptop)
    • Set the domain to home.arpa.
    • Provide the IP addresses on the local LAN (for example, in my setup, I set 192.168.60.1 for the router and 192.168.60.254 for my host).
    • Make sure to Apply for your changes to take effect
  • To verify this configuration, navigate to Interfaces > Diagnostics > Ping and type the hostnames (i.e. laptop and opnsense) and run the job to make sure that the ping operation is successful. You can stop and delete the jobs after your verification is complete.
    • Notice: Windows Firewall blocks all ICMP packets by default, so you may not be able to PING your Windows 11 host. You do not need to modify your firewall settings just for this test.

Create Documentation and Backups

This is a good time to create documentation for yourself. For example, record your host-only and LAN network configurations, such as IP addresses and DHCP address ranges. In case you have to re-create this environment in the future, this document should provide all the information you will need. A simple plain text file will do.

Backup your OPNsense configuration by navigating to System > Configuration > Backup. You will get an XML-formatted configuration that you are able to use to restore to OPNsense configurations later if something goes wrong. Repeat this process every time you make configuration changes.

After this lab, and before starting the next labs, it is a good idea to perform a full backup of your virtual machines. Start by powering off the VMs by shutting them down from their OS so they are powered off properly. For OPNsense, navigate to Power > Power Off to turn off the system (do this every time you are done with your lab). Once all VMs are turned off, exit your virtualization software (exit VirtualBox). Navigate to where your VMs are stored and back up the VMs by adding the entire VM directory to a compressed file. You can store this backup in an external location, such as removable storage or a cloud service.

Notice: Do not store your VM directory in a location that will be synced with OneDrive. Experience shows synchronizing VM files with OneDrive may result in corruption of your VMs. You can back up the compressed files to OneDrive if desired.

You can perform additional backups using the snapshot function through your virtualization software, but periodic full backups are recommended.
