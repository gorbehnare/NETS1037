Lab 06 - NetFlow

In this lab, you will enable NetFlow on your OPNsense router and observe the results. Additionally, you will install ntopng on your nmshost and explore the web application UI. Start your opnsense, loghost, and nmshost for this lab.

Enable NetFlow on OPNsense

  • Visit Reporting > NetFlow and configure NetFlow as per OPNsense help on the page. Select all your interfaces to be monitored and specify the WAN interface to avoid duplicate entries. For now, only select the Capture local option.
  • Navigate to the Reporting > Insight, and you may or may not see any activity yet. Keep this page open, and from your other systems, such as loghost and/or snmhost, generate some traffic. For example, check for and install updates using sudo apt update and sudo apt upgrade.
  • wait for a minute or two so that the Insight page Totals is updated (you can refresh the page to see changes)

The amount of traffic passing through the router depends on how many updates are available to download.

The ntopng Package on nmshost

On your nmshost, follow the documentation to add the ntop package repository and install the current stable version of ntopng community edition in your system:

  • visit https://www.ntop.org/support/documentation/software-installation/
  • Make sure to select the stable branch of the community edition of ntopng.

There may be differences between OS versions, but this is what it looks like on Ubuntu 24.04 LTS:

# Make sure you are installing on `nmshost`
ssh nmshost

# use root shell, or use sudo
sudo bash

apt-get install software-properties-common wget
add-apt-repository universe
wget https://packages.ntop.org/apt-stable/24.04/all/apt-ntop-stable.deb
apt install ./apt-ntop-stable.deb

apt update
apt install ntopng
ufw allow 3000/tcp
# Make sure you are installing on `nmshost`
ssh nmshost

# use root shell, or use sudo
sudo bash

apt-get install software-properties-common wget
add-apt-repository universe
wget https://packages.ntop.org/apt-stable/24.04/all/apt-ntop-stable.deb
apt install ./apt-ntop-stable.deb

apt update
apt install ntopng
ufw allow 3000/tcp

Once the installation is completed:

  • In a web browser from your host, navigate to [http://nmshost:30000]
  • Use admin/admin as the default, and change the password as required.
  • Generate some traffic to and from nmshost, ping an internet IP, run update, ssh into the system, or log into LibreMNS on this system.
    • You can navigate to Flows and watch Live Flows, and watch various activities such as SSH, SNMP and other traffic that you generate show up in the list.
    • Please note that it may take some time for many fields to be allocated based on your system activity. The longer you have your lab environment running, the more activity and interaction between hosts you will observe in your network.
  • Once you have some activity logged, navigate back to your Dashboard > Traffic and select your local NIC from the drop-down menu at the top to view network activity (for example, the NIC name is enp0s3 in my system).

The ntopng plugin can be installed on OPNsense, and you can forward NetFlow traffic from OPNsense to your nmshost for collection. The documentation is located under Third-Party Integrations / OPNsense. You are not required to perform this step in your lab. If you are curious and would like to play, I recommend cloning your systems or performing a full backup so you can restore the current state of your systems before you experiment with these options. There are many configurations and fine-tuning that can be done on ntopng, and some features are available only if you pay for a commercial license.
