Lab 03 - The Log Analysis System

In this lab, we are creating a webhost server to allow for the separation of the analysis of logs from the capture of logs. You should have completed the previous labs and have a working Host-only network with OPNsense router as gateway and loghost logging server.

DNS Overrides on OPNsense

Before you begin, start your opnsense router and log into the web UI and add the following DNS override:

  • Navigate to: Services > Unbound DNS > Overrides
  • Add the following overrides:
    • Host: webhost
    • Domain: home.arpa
    • IP Address: Your Host-only network with host address 4, for example: 192.168.60.4
    • Any Description for your reference, for example, The Webhost Server

VM Creation and OS installation

We will be using this webhost for a couple of labs. Please keep in mind that you will have your opnsense router and loghost running at the same time, so make sure to monitor your host resources. Exit background tasks and un-needed programs on your host computer to free up resources if needed.

Create a new VM for Ubuntu Server and attach the NIC to your Host-only Network.

  • Start with at least 2 CPUs
  • Start with at least 1GB
  • Set the disk to be at least 20GB. You can resize this later if needed; however, additional steps will be needed to resize the partition and volume in Linux.

Install Ubuntu Server LTS (download the latest .iso image from ubuntu.com) with the following configuration during setup:

  • Configure your NIC with Manual Configuration:

    • Use the static IP address from your network with the host address set to 4 (for example, 192.168.60.4)
    • The Gateway and Name Server should be set to your OPNsense router IP Address you had created in the previous lab (for example, 192.168.60.1)
    • Use the search domain home.arpa

  • Use the entire disk (can use the default settings)

  • Under Profile Configuration, enter the following:

    • Set Your server name (system hostname) to webhost
    • Add a user account for yourself (your first name) and configure a password

  • Under SSH Configuration choose to Install OpenSSH server

  • You do NOT need to install any additional software or packages.

  • Once the installation is complete, reboot the server and disconnect the installation .iso image from the optical drive and hit return when prompted.

If your environment is configured correctly and installation is successful, you should be able to log in through console or SSH, and your server should have access to the internet for downloading and installing updates through your OPNsense router. You can attempt to update and upgrade your system using sudo apt update && sudo apt upgrade -y.

Modify webhost to send logs to loghost

  • Ensure that your loghost is started and running.
  • Ensure that your DNS configurations are correct by attempting to ping loghost from webhost (or vice versa). The correct .3 and .4 IP addresses should be resolved automatically.
  • Modify rsyslog configuration on the webhost to send logs to the loghost:
    • Modify /etc/rsyslog.conf and add the following configuration line:
    # 
# Send logs to our loghost rsyslog server on port 514
#
# For UDP use: 
#*.* @loghost:514
# For TCP use:
#*.* @@loghost:514
# Just use the UDP protocol: 
*.* @loghost:514

    • Restart rsyslog service: sudo systemctl restart rsyslog
  • On your loghost, verify that webhost logs are being received and stored in the MySQL Database:
# Make sure that the entries are greater than 0 and are increasing
user@loghost:~$ sudo mysql <<<"select count(*) from Syslog.SystemEvents where FromHost = 'webhost';"

MySQL Account for webhost

On loghost, allow remote database access for webhost through the UFW firewall:

sudo ufw allow from 192.168.xxx.4 to 192.168.xxx.3 port 3306 proto tcp
sudo ufw allow from 192.168.xxx.4 to 192.168.xxx.3 port 3306 proto tcp

Modify the MySQL service to allow remote connections.
Modify /etc/mysql/mysql.conf.d/mysqld.cnf:

bind-address        = 0.0.0.0
bind-address        = 0.0.0.0

You must restart the service using sudo service mysql restart for changes to take effect. Also, verify the changes using the sudo ss -tlpn | grep mysql to make sure that the service is listening from all IP addresses.

Exploring Logs Directly from MySQL

Connect from webhost to MySQL on loghost. First, you will need to install MySQL Client:

sudo apt install mysql-client
sudo apt install mysql-client

From webhost test your MySQL access:

user@webhost:~$ mysql -u webhost -p -h loghost <<<"select count(*) from Syslog.SystemEvents;"
Enter password: 
count(*)
45035
user@webhost:~$ mysql -u webhost -p -h loghost <<<"select count(*) from Syslog.SystemEvents;"
Enter password: 
count(*)
45035

You can use SQL directly to view and sort logs from the MySQL database. For example, to pull all logs for your webhost and export to a .csv file so you can open it in any spreadsheet program:

# Export all the fields and entries where the "FromHost" field is = 'webhost' 
# to a file named "webhost_logs.csv":
mysql -u webhost -p -h loghost <<<"select * from Syslog.SystemEvents where FromHost = 'webhost';" > ~/webhost_logs.csv
# Export all the fields and entries where the "FromHost" field is = 'webhost' 
# to a file named "webhost_logs.csv":
mysql -u webhost -p -h loghost <<<"select * from Syslog.SystemEvents where FromHost = 'webhost';" > ~/webhost_logs.csv

You can copy the file to your desktop using the scp from your host computer using:

scp user@192.168.xxx.3:~/webhost_logs.csv ./webhost_logs.csv
scp user@192.168.xxx.3:~/webhost_logs.csv ./webhost_logs.csv

You can open webhost_logs.csv using Excel, LibreOffice Sheets, or any other software to view the contents (even Notepad). You can also experiment with specifying only the fields and contents you are interested in. For example, attempt to export the same file, but only select ID, ReceivedAt, DeviceReportedTime, Facility, Priority, FromHost, and Message. You can also play with other options to look for specific logs. Here are some examples for you to explore and play with:

mysql -u webhost -p -h loghost <<<"select ID,fromhost,receivedat,devicereportedtime,facility,priority,message from Syslog.SystemEvents where FromHost = 'opnsense.home.arpa';" > opnsense.csv

mysql -u webhost -p -h loghost <<<"select ID,fromhost,receivedat,devicereportedtime,facility,priority,message from Syslog.SystemEvents;" > all_logs.csv
mysql -u webhost -p -h loghost <<<"select ID,fromhost,receivedat,devicereportedtime,facility,priority,message from Syslog.SystemEvents where FromHost = 'opnsense.home.arpa';" > opnsense.csv

mysql -u webhost -p -h loghost <<<"select ID,fromhost,receivedat,devicereportedtime,facility,priority,message from Syslog.SystemEvents;" > all_logs.csv

Web-based Log Analysis using LogAnalyzer

The loganalyzer is a PHP-MYSQL-based application. Install the mysql-server package on webhost to use for this application, and create a MySQL user account:

sudo apt install mysql-server
sudo mysql -h localhost -u root <<EOF
create database loganalyzer;
create user 'loganalyzer'@'localhost' identified by 'loganalyzer';
grant all on loganalyzer.* to 'loganalyzer'@'localhost';
flush privileges;
EOF
exit
sudo apt install mysql-server
sudo mysql -h localhost -u root <<EOF
create database loganalyzer;
create user 'loganalyzer'@'localhost' identified by 'loganalyzer';
grant all on loganalyzer.* to 'loganalyzer'@'localhost';
flush privileges;
EOF
exit

Download LogAnalyzer from Adison v4.1.12 (stable) or use alternate downloads and installation instructions source: https://github.com/rsyslog/loganalyzer.

The LogAnalyzer requires apache and php packages. Install and configure loganalyzer using the following method:

sudo su
cd ~
# Download the loganalyzer files:
wget http://download.adiscon.com/loganalyzer/loganalyzer-4.1.13.tar.gz

# Extract and copy the files to web document root
tar zxf loganalyzer-4.1.13.tar.gz
mkdir -p /var/www/html/loganalyzer
cp -r loganalyzer-4.1.13/src/ /var/www/html/loganalyzer

# Install Apache webserver and PHP modules
sudo apt install apache2 php php-gd php-mysql libapache2-mod-php

# restart the apache2 service
systemctl restart apache2

# Verify that the files are in the correct path
cd /var/www/html/loganalyzer
ls -la

# Create the loganalyze configuration file
touch /var/www/html/loganalyzer/config.php

# give read and execute permissions to group and others
chmod 755 ./* -R
chown www-data:www-data -R

From your host computer, open a web browser and navigate to the following URL:
http://<webhost_IP>/loganalyzer/install.php. Follow the instructions on screen to complete the configuration of loganalyzer:

  • You may need to change permissions for /var/www/html/loganalyzer/config.php to 666 first to allow configuration to be written to the file.
  • Under Basic Configuration, select Yes on the Enable User Database
  • Enter your MySQL credentials you created earlier for loganalyzer.
  • Require users to log in
  • Proceed with the steps through the wizard to create tables and check SQL results
  • Create a new user account for your Web access
  • Under Create the First Source for syslog message:
    • Specify a name for your source (for example, loghost-rsyslog-mysql)
    • Source Type: MySQL Native
    • Select View: Syslog Fields
    • Table type: MonitorWare
    • Database Host: loghost (should have a working DNS, or may have to enter the IP address for loghost)
    • Database Name: Syslog
    • Database Tablename: SystemEvents
    • Database User: webhost
    • Database Password: ****Your_MySQL_Password****
      You should be able to finish the wizard and see the log-in page. You can see all logs, search for keywords, or choose to sort by any desired field, such as severity.

Tips for Troubleshooting LogAnalyzer

If you are experiencing any issues, such as blank pages or internal errors with any PHP-based application, your best starting point is to look at the Apache server logs located at /var/log/apache2/error.log. You can open a new terminal and use tail -f to watch the live logs as you navigate to different web pages:

sudo tail -f /var/log/apache2/error.log 
sudo tail -f /var/log/apache2/error.log

UFW Firewall for webhost

Currently, the only services that should be accessible on your system are SSH and HTTP. Configure your UFW to allow ports 22 and 80 and enable the firewall.

sudo ufw allow ssh
sudo ufw allow http
sudo ufw enable

# Check status:
sudo ufw status
sudo ufw allow ssh
sudo ufw allow http
sudo ufw enable

# Check status:
sudo ufw status
