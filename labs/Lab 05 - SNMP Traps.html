Lab 05 - SNMP Traps

In this lab, you will configure SNMP traps to be sent from your lab systems and handled by nmshost. Additionally, we are adding other forms of notification to nmshost using local email. We will configure webhost and loghost to send SNMP traps to `nmshost. Finally, we will enable alerts on LibreNMS. Before you begin, make sure your lab environment is up and running, and you have completed the previous labs successfully.

Setup Local-Only Email on nmshost

Install postfix and mailutils so we can test sending emails when traps are received. When prompted:

  • Install it as a Local Only site
  • Set hostname for the mailserver to nmshost.home.arpa
sudo apt install postfix mailutils

Configure nmshost to Send and Accept SNMP Traps

On your nmhost, ensure snmpd and snmptrapd are installed, and modify the configurations as follows (see the script below):

  • Update MIBs, including commenting out the MIBs line in /etc/snmp.conf
  • In the /etc/snmp/snmpd.conf, the agentAddress should have already been configured to allow remote connections from the previous lab
  • Configure /etc/snmp/snmpd.conf to send the default set of traps to nmshost, and create internalUser for querying the data sources and specifying the default traps
  • Change the Ubuntu default service unit settings for the snmpd service startup to not exclude the mteTrigger needed for traps in /lib/systemd/system/snmpd.service, by removing those options from the startup command (remove -I -smux,mteTrigger,mteTriggerConf -f)
  • reload the systemd daemon to apply system configuration changes

The following script should accomplish these tasks:

# update your system, just in case
sudo apt update && sudo apt upgrade

# Install the appropriate software packages
sudo apt install snmpd snmptrapd snmp-mibs-downloader

# Modify snmp.conf to enable MIBs by commenting out the line
sudo sed -i -e 's/(^MIBs :.*)/#\\1/' /etc/snmp/snmp.conf

# modify the SNMP daemon configuration to add users and a destination for SNMP traps
sudo grep -i internaluser /etc/snmp/snmpd.conf || cat <<< "

#internal use user for querying trap data
createUser internalUser SHA fubar2snafu AES
rouser        trapguy
iquerySecName trapguy

# Activate the standard monitoring entries
defaultMonitors         yes
linkUpDownNotifications yes

# send traps to nmshost using community public
trap2sink nmshost public

# add an example custom trap to watch for high CPU usage
monitor -r 5 machineTooBusy hrProcessorLoad > 75

"| sudo tee -a /etc/snmp/snmpd.conf

# Remove the mteTrigger exclusions from the start command in the service using a file
sudo sed -i 's/-I -smux,mteTrigger,mteTriggerConf //' /lib/systemd/system/snmpd.service

# Reload systemd to apply the new system settings
sudo systemctl daemon-reload

Configure SNMP Trap Handling in nmshost

Add the public authCommunity with log and execute in /etc/snmp/snmptrapd.conf. This will set up the traps to get logged and execute the trap handler to send an email to yourself:

# Inside /etc/snmp/snmptrapd.conf add:
authCommunity log,execute public
# make sure to replace the <username> with your own user account
traphandle default /usr/bin/traptoemail -s localhost <username>@nmshost.home.arpa
# Inside /etc/snmp/snmptrapd.conf add:
authCommunity log,execute public
# make sure to replace the <username> with your own user account
traphandle default /usr/bin/traptoemail -s localhost <username>@nmshost.home.arpa

Modify the default Ubuntu service unit settings for service startup to send logs to syslog in /lib/systemd/system/snmptrapd.service, by replacing the -LOw with -Lsd flags:

[Unit]
Description=Simple Network Management Protocol (SNMP) Trap Daemon.

[Service]
Type=notify
User=Debian-snmp
ExecStart=/usr/sbin/snmptrapd -Lsd -f udp:162 udp6:162
ExecReload=/bin/kill -HUP $MAINPID

Allow traps through the nmshost firewall on port 162/udp:

sudo ufw allow 162/udp
sudo ufw allow 162/udp

Reload systemd configurations and start snmptrapd service.

sudo systemctl daemon-reload
sudo systemctl start snmptrapd

Configure LibreNMS to show alerts

LibreNMS has an alerting capability that can use rules to match events that get recorded and then raise their visibility by putting them into an alert widget or by sending those alerts to a distribution mechanism. To demonstrate alerts, we will set up some simple ones to show in the standard alerts widget.

  1. Verify that the LibreNMS trap handler script is in your snmptrapd.conf file on nmshost
sudo grep -i librenms /etc/snmp/snmptrapd.conf || cat <<< "
traphandle default /opt/librenms/snmptrap.php
" | sudo tee -a /etc/snmp/snmptrapd.conf
sudo grep -i librenms /etc/snmp/snmptrapd.conf || cat <<< "
traphandle default /opt/librenms/snmptrap.php
" | sudo tee -a /etc/snmp/snmptrapd.conf
  1. Navigate to Devices, click on any host and click on the gear icon to modify.
  2. Use the Alert Rules page to add the default rules (should be a large button for this if there are no alert rules added). A list of default alert types will be added to the monitored devices.
  3. Navigate to the dashboard, and add an Alerts widget to the dashboard.
  4. Try viewing and acknowledging any alerts you may see (there may not be any).
