Introductory Information

Resource Requirements

The ELK stack is written in Java and requires the Java Runtime Environment. The official recommended minimum resource requirements for this stack are at least 4 CPU cores and 16GB of RAM. Elasticsearch service consumes more than 4GB of RAM on its own, while in my setup, Logstash and Kibana consumed 1GB and 2GB of RAM, respectively. I recommend at least 8GB of RAM for this VM. Additionally, to ensure that sufficient disk space is available, start with at least 20GB of storage. You can always monitor resources using htop, free, and df and take appropriate actions to resize your VM as required.

Service Start and Stop Order

If the pages do not load and services need to be restarted, the services need to be started in the correct order:

  1. Elasticsearch (Back-end Database)
  2. Logstash (gathering and importing data into Elasticsearch)

  • /var/log/elasticsearch/elasticsearch.log
  • /var/log/kibana/kibana.log
  • /var/log/logstash/logstash-plain.log

  • Kibana Reference Documentation: https://www.elastic.co/docs/reference/kibana
  • Logstash Reference Documentation: https://www.elastic.co/docs/reference/logstash

In the official documentation, the path for all configurations and utilities uses a relative path (starting with ./) for all sub-commands and utilities. To save you the time, effort, and confusion, the following are the full paths for each application. Replace the ./ from the documentation with these for each one, or cd to the path:

  • Kibana server path: /usr/share/kibana/

Shutdown Sequence for webhost with ELK

Please note that the Logstash service stop times vary widely based on what the system is doing at the time the shutdown command is issued. If you kill the process or forcefully stop the service, you may end up with some corrupted or incomplete entries in the database. If you wait long enough, it will eventually shut down. The following is the procedure for a controlled shutdown for your host:

  • Stop the logstash service first: sudo service logstash stop (may take a long time)
  • Stop kibana service: sudo service kibana stop
  • Stop the elasticsearch service: sudo service elasticsearch stop
  • You should be able to issue the sudo poweroff to power off the loghost

Note On Updating/Upgrading the ELK stack on webhost

If you are performing an update on your system using sudo apt update and sudo apt upgrade, read the list of packages that are being installed before you proceed with y for yes to the upgrade.

  • If an update for the ELK packages are available, you should see that all 3 of them listed. Since these are Java-based applications, the version numbers for elasticsearch, kibana, and logstash must match.
  • I highly recommend that you stop the three (3) services using the procedure above before proceeding with the upgrade! As always, backup your VMs periodically.

Agent-based Monitoring (Optional)

Some logging and monitoring systems install agents in order to extend functionality. The following is an example of a script to collect some telemetry information from log files and create a separate log file (sources: Blog post at https://blog.cyberdesserts.com/elk-stack-security-monitoring-tutorial/ and scripts from GitHub at https://github.com/cyberdesserts/elk_stack/tree/main/scripts/telemetry).

  • Create the script file and store somewhere in your system (for example, you can use /root/linuxtelemetery.sh)
  • Use cron entry to run the script every few min
    • Use sudo crontab -e and add an entry, for example: */5 * * * * /root/linux_telemetry.sh 192.168.60.3 514 >> /var/log/telemetry.log 2>&1
#!/bin/bash

# Linux Cybersecurity MVP Telemetry Script
# Works with Ubuntu, Debian, CentOS, RHEL, and other systemd distributions
#
# Usage: ./linux_telemetry.sh [ELK_HOST] [PORT]
# Examples:
#   ./linux_telemetry.sh localhost 514      # ELK on same machine
#   ./linux_telemetry.sh 192.168.1.100 514  # ELK on remote server

HOST="${1:-localhost}"
PORT="${2:-514}"
HOSTNAME=$(hostname -s)

function send_syslog() {
    local program=$1
    local message=$2
    timestamp=$(date '+%b %d %H:%M:%S')
    syslog_msg="<110>$timestamp $HOSTNAME $program: $message"
    echo "$syslog_msg" | nc -u -w 1 $HOST $PORT 2>/dev/null
    echo "Security telemetry: $program - $message"
}

echo "Collecting Linux cybersecurity telemetry..."
echo "Sending to ELK stack at $HOST:$PORT"
echo ""

# 1. Failed Login Attempts
if [ -f /var/log/auth.log ]; then
    failed_logins=$(grep -c "Failed password\|authentication failure" /var/log/auth.log 2>/dev/null)
elif [ -f /var/log/secure ]; then
    failed_logins=$(grep -c "Failed password\|authentication failure" /var/log/secure 2>/dev/null)
else
    failed_logins=$(journalctl -u sshd --since "1 hour ago" 2>/dev/null | grep -c "Failed password" || echo "0")
fi
send_syslog "auth-monitor" "Failed login attempts: $failed_logins"

# 2. Active Network Connections
active_connections=$(ss -tun state established 2>/dev/null | wc -l)
active_connections=$((active_connections > 0 ? active_connections - 1 : 0))
send_syslog "network-monitor" "Active TCP/UDP connections: $active_connections"

# 3. Processes with Network Activity
network_processes=$(ss -tunp 2>/dev/null | awk 'NR>1 {print $7}' | cut -d'"' -f2 | sort -u | wc -l)
send_syslog "process-monitor" "Processes with network activity: $network_processes"

# 4. New Files in /tmp
tmp_files=$(find /tmp -type f -mmin -60 2>/dev/null | wc -l)
send_syslog "file-monitor" "New files in /tmp last hour: $tmp_files"

# 5. System Load
load_avg=$(cat /proc/loadavg | awk '{print $1}')
send_syslog "load-monitor" "1-minute load average: $load_avg"

#!/bin/bash

# Linux Cybersecurity MVP Telemetry Script
# Works with Ubuntu, Debian, CentOS, RHEL, and other systemd distributions
#
# Usage: ./linux_telemetry.sh [ELK_HOST] [PORT]
# Examples:
#   ./linux_telemetry.sh localhost 514      # ELK on same machine
#   ./linux_telemetry.sh 192.168.1.100 514  # ELK on remote server

HOST="${1:-localhost}"
PORT="${2:-514}"
HOSTNAME=$(hostname -s)

function send_syslog() {
    local program=$1
    local message=$2
    timestamp=$(date '+%b %d %H:%M:%S')
    syslog_msg="<110>$timestamp $HOSTNAME $program: $message"
    echo "$syslog_msg" | nc -u -w 1 $HOST $PORT 2>/dev/null
    echo "Security telemetry: $program - $message"
}

echo "Collecting Linux cybersecurity telemetry..."
echo "Sending to ELK stack at $HOST:$PORT"
echo ""

# 1. Failed Login Attempts
if [ -f /var/log/auth.log ]; then
    failed_logins=$(grep -c "Failed password\|authentication failure" /var/log/auth.log 2>/dev/null)
elif [ -f /var/log/secure ]; then
    failed_logins=$(grep -c "Failed password\|authentication failure" /var/log/secure 2>/dev/null)
else
    failed_logins=$(journalctl -u sshd --since "1 hour ago" 2>/dev/null | grep -c "Failed password" || echo "0")
fi
send_syslog "auth-monitor" "Failed login attempts: $failed_logins"

# 2. Active Network Connections
active_connections=$(ss -tun state established 2>/dev/null | wc -l)
active_connections=$((active_connections > 0 ? active_connections - 1 : 0))
send_syslog "network-monitor" "Active TCP/UDP connections: $active_connections"

# 3. Processes with Network Activity
network_processes=$(ss -tunp 2>/dev/null | awk 'NR>1 {print $7}' | cut -d'"' -f2 | sort -u | wc -l)
send_syslog "process-monitor" "Processes with network activity: $network_processes"

# 4. New Files in /tmp
tmp_files=$(find /tmp -type f -mmin -60 2>/dev/null | wc -l)
send_syslog "file-monitor" "New files in /tmp last hour: $tmp_files"

# 5. System Load
load_avg=$(cat /proc/loadavg | awk '{print $1}')
send_syslog "load-monitor" "1-minute load average: $load_avg"

# 6. SSH Sessions (Linux-specific)
ssh_sessions=$(who | grep -c pts/ 2>/dev/null || echo "0")
send_syslog "session-monitor" "Active SSH sessions: $ssh_sessions"

# 7. Sudo Usage
if [ -f /var/log/auth.log ]; then
    sudo_commands=$(grep -c "sudo:" /var/log/auth.log 2>/dev/null || echo "0")
elif [ -f /var/log/secure ]; then
    sudo_commands=$(grep -c "sudo:" /var/log/secure 2>/dev/null || echo "0")
else
    sudo_commands=$(journalctl --since "1 hour ago" 2>/dev/null | grep -c "sudo:" || echo "0")
fi
send_syslog "privilege-monitor" "Sudo commands in logs: $sudo_commands"

# 8. Listening Ports
listening_ports=$(ss -tlun 2>/dev/null | wc -l)
listening_ports=$((listening_ports > 0 ? listening_ports - 1 : 0))
send_syslog "port-monitor" "Listening ports: $listening_ports"

echo ""
echo "Linux telemetry sent to ELK stack at $HOST:$PORT"
