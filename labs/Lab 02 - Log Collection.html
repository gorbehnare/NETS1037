Lab 02 - Log Collection

In this lab, you are creating a loghost logging server using Rsyslog and MySQL for centralized log collection and storage. You should have completed the previous labs and have a working Host-only network with OPNsense router as the gateway. You will also configure OPNsense and other hosts to send logs to this system.

You can refer to rsyslog documentation for basic configuration, operation, and troubleshooting at: https://docs.rsyslog.com/doc/getting_started/beginner_tutorials/index.html.

The loghost Logging Server

Before you begin, start your opnsense router and log into the web UI and add the following DNS override:

  • Navigate to: Services > Unbound DNS > Overrides
  • Add the following overrides:
    • Host: loghost
    • Domain: home.arpa
    • IP Address: Your Host-only network with host address 3, for example: 192.168.60.3
    • Any Description for your reference, for example, The Loghost Server

VM Creation and OS Installation

Create a new VM for Ubuntu Server and attach the NIC to your Host-only Network.

  • The minimum resource requirements are 1 CPU (2 CPU cores are recommended) and a minimum of 1024 MB of RAM

Install Ubuntu Server LTS (download the latest .iso image from ubuntu.com) with the following configuration:

  • Configure your NIC with Manual Configuration:
    • Use the static IP address from your network with the host address set to 3 (for example, 192.168.60.3)
    • The Gateway and Name Server should be set to your OPNsense router IP Address you had created in the previous lab (for example, 192.168.60.1)
    • Use the search domain home.arpa

  • Use the entire disk
  • Under Profile Configuration, enter the following:
    • Set Your server name (system hostname) to loghost
    • Add a user account for yourself (your first name) and configure a password
  • Under SSH Configuration choose to Install OpenSSH server
  • You do NOT need to install any additional software or packages.
  • Once the installation is complete, reboot the server and disconnect the installation .iso image from the optical drive and hit return when prompted.

If your environment is configured correctly and installation is successful, you should be able to log in, and your server should have access to the internet for downloading and installing updates through your OPNsense router.

Sending Logs to the Log Server

OPNsense Logs

  • Log into the OPNsense Web UI and navigate to System > Settings > Logging
  • Add your loghost as a remote logging server by visiting the Remote tab and clicking on the +.
    • If you have configured your DNS overrides, you should be able to use loghost instead of the IP address

On your loghost server, ensure that the logs are arriving from other hosts by querying the database. The following are examples:

sudo mysql <<<"select count(*) from Syslog.SystemEvents where FromHost != 'loghost';"
sudo mysql <<<"select count(*) from Syslog.SystemEvents where FromHost = 'opnsense.home.arpa';"
sudo mysql <<<"select count(*) from Syslog.SystemEvents where FromHost != 'loghost';"
sudo mysql <<<"select count(*) from Syslog.SystemEvents where FromHost = 'opnsense.home.arpa';"

Tip: The following is another example to see what hosts other than loghost exist in the database. This is useful for troubleshooting, or you want to see the hostnames. Feel free to try out other SQL commands to view the logs.

sudo mysql <<<"select FromHost from Syslog.SystemEvents where FromHost != 'loghost';" | more
sudo mysql <<<"select FromHost from Syslog.SystemEvents where FromHost != 'loghost';" | more

Windows Host (Optional)

Create a VM for Windows Desktop

  1. Create a new VM and connect to your Host-only Network
    • Windows requires at least 2 CPUs and 4GB of RAM
  2. Obtain and attach an installation ISO for Windows 10/11 - see microsoft.com
  3. Install the OS. The default install is fine for our purposes, evaluation mode is fine, no need to purchase a license
  4. Select a static IP address for your Windows VM, and add an entry to the DNS Resolver overrides under Services > Unbound DNS > Overrides in your OPNsense router to create the name windows.home.arpa with the address of your Windows VM

Modify the Windows desktop VM to send logs to loghost using rsyslog

  1. Download an agent program to send Windows Event Logs to a syslog server. Examples:
    • rsyslog Windows Agent and (documentation)
    • nxlog (30-day trial) (requires providing an email address and creating an account).
    • Find and choose any other Windows syslog agent you prefer
  2. Install the agent and configure it to send the logs to loghost.
  3. Confirm windows log messages are showing up in the text log files on loghost
sudo grep windows /var/log/syslog|head
sudo mysql <<<"select count(*) from Syslog.SystemEvents where FromHost = 'windows.home.arpa';"
sudo grep windows /var/log/syslog|head
sudo mysql <<<"select count(*) from Syslog.SystemEvents where FromHost = 'windows.home.arpa';"
